# magi-plugins

Official plugin registry and plugin source repository for [Magi](https://github.com/asukaonly/magi).

This is a **companion repository** to the [Magi](https://github.com/asukaonly/magi) main repository. The main repo contains the core framework, desktop app, and two bundled core plugins (`core-tools`, `core-actions`). All other optional plugins live here and are installed on demand via the in-app plugin marketplace.

## How It Works

```
Magi App  в†’  Settings в†’ Extensions в†’ Marketplace
                          в†“
              Fetches registry.json from this repo
                          в†“
              Shows available plugins with install/update buttons
                          в†“
              git clone --depth 1 --sparse (this repo)
                          в†“
              Copies plugin to ~/.magi/plugins/<plugin_id>/
```

1. The Magi backend fetches `registry.json` from this repo's `main` branch.
2. The marketplace UI displays available plugins with metadata from the registry.
3. When the user clicks **Install**, the backend does a shallow sparse checkout of this repo, extracts the target plugin directory, and copies it to `~/.magi/plugins/<plugin_id>/`.
4. The plugin manager rescans and loads the new plugin.

## Usage

### Installing plugins (end users)

Open the Magi desktop app в†’ **Settings в†’ Extensions в†’ Marketplace** в†’ click **Install** on any plugin.

### Regenerating registry.json

After adding or updating a plugin, regenerate the registry index:

```bash
python scripts/build-registry.py
```

This reads all `plugin.toml` files under `plugins/` and writes `registry.json`. Commit the updated `registry.json` along with your plugin changes.

### Adding a new plugin

1. Create a new directory under `plugins/`.
2. Add `plugin.toml` and `plugin.py` (see the plugin development guide in the [main repo docs](https://github.com/asukaonly/magi/blob/main/docs/plugin-development-guide.md)).
3. Run `python scripts/build-registry.py` to update `registry.json`.
4. Commit and push.

### Local development

For development testing, symlink or copy your plugin directory to `~/.magi/plugins/<plugin_id>/`, then rescan from the Magi settings page.
